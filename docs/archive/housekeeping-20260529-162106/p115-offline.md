# 115离线下载

此文档只处理已判定为离线下载类的链接。链接分流总规则见 `p115-routing.md`；本文件不重复定义分流规则。

## 触发场景
- 少爷直接发来 `ed2k` 链接。
- 少爷直接发来磁力链接。
- 少爷直接发来其他可离线处理的下载直链，且不是 `115` 分享链接。
- 少爷明确要求“用 115STRM 的离线下载”。

## 固定链路
1. 直接执行：`/ol <链接>`。
2. 以命令触发成功作为已提交依据；需要补验收时，再看插件返回消息、消息表或日志。

## 关键规则
- 只要链接已经判定为离线下载类，直接把链接交给 `/ol` 处理，不做多余前置校验或人工介入。
- 只有少爷明确指定普通下载器时，才改走 qBittorrent / Transmission。
- `ed2k` 链接优先使用完整标准格式；当前已跑通的格式保留结尾 `|/`。
- `115` 分享链接不走 `/ol`；默认改走 `/p115_add_share`，若少爷明确要求分享直生 `STRM`，则走 `/p115_share_strm`。
- 若后续出现“未识别到媒体信息，无法入库！”，只代表该文件不能自动按影视条目识别整理，不等于离线任务提交失败。
- 若失败，优先检查：
  - 链接格式是否完整。
  - `P115StrmHelper` 是否运行。
  - `/ol` 是否仍在插件能力列表中。
  - `moviepilot.log`、`p115strmhelper.log` 与消息表反馈。
