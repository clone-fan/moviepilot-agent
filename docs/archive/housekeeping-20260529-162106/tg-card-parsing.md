# TG资源卡片解析

此文档用于统一处理 Telegram 转发影视/音乐/合集卡片中的真实资源入口识别规则。

## 总原则
- 先从卡片正文里找真实资源入口，再决定是否需要继续打开 `查看资源`。
- 推广链接、机器人入口、频道入口、公费服、机场、群组、投稿入口都不是资源入口。
- 能直接拿到真实链接时，不额外多绕一层。
- 以速度优先：正文已含真实资源链接时，直接进入 115 分流，不做页面验证。

## 主判定顺序
1. 先看正文是否已给出真实资源链接。
2. 若正文只有 `查看资源(telegra.ph)`，直接按同路径改走 `te.legra.ph` 抓取，不再先试 `telegra.ph`。
3. 若 `Telegraph` 页面仍不是最终资源，而是外部清单链接，再继续下钻。

## 正文直取规则
- 若正文已出现 `链接:` / `链接：`，且值为 `115cdn` / `115.com` / `magnet` / `ed2k` 等真实资源链接，直接认定为实际资源入口。
- 若正文为 `链接：点击跳转(真实链接)`，提取括号中的真实链接，`点击跳转` 只是展示文案。
- 若正文同时给出 `夸克` / `百度` / `迅雷` / `115` 等多网盘直链，均视为真实资源入口。
- 默认优先提取 `115` 链接用于本地工作流，其余链接只作补充来源保留。
- 若正文里的 `转存到115网盘(...)` 与主 `链接` 指向同一个 `115` 地址，视为同一资源，不重复输出。

## Telegraph 解析规则
- 若正文只有 `查看资源(telegra.ph)`，默认直接改走 `te.legra.ph` 同路径镜像。
- 若 `telegra.ph` / `api.telegra.ph` 超时，只在镜像未取到时才考虑强制解析到可达 IP。
- 解析时优先做轻量正文抽取，只抓真实资源链接，不先做整页验证。
- 若 `Telegraph` 页面里有唯一外链，且该外链是 `115cdn` / `115.com` / `magnet` / `ed2k` 清单页等真实资源入口，则直接输出该链接。
- 若 `Telegraph` 正文未放超链接，而是直接写出 `ed2k://...` / `magnet:?` 原文，则直接提取整段协议链接。
- `资源链接（长按复制）`、`查看链接` 这类文案只是提示，不是噪音。

## 外部清单下钻
- 若 `Telegraph` 指向 `files.catbox.moe/*.txt` 之类文本清单，需继续下钻读取清单内容。
- 这类 `catbox` 文本链接本身应视为真实资源清单入口，不能误判为空页。

## 元数据忽略规则
- `电视剧：片名 - SxxExx`、`TMDB ID`、评分、分类、质量、大小、主演、简介等元数据只用于识别内容，不影响资源入口判定。
- `音乐/明星` 卡片中的生日、出生地、代表作等字段同样只用于识别内容。
- `None`、空值、`暂无` 这类占位字段直接忽略。

## 噪音排除
- 机场、公费服、Emby、投稿机器人、搜索机器人、频道、群组、VIP、广告入口、一键转存入口都不是资源入口。
- 同一真实链接的重复包装只保留一份。

## 已验证模式
- TG正文直给 `115` 分享链接。
- TG正文 `点击跳转(115链接)`。
- TG正文多网盘并列，默认优先 `115`。
- `查看资源 -> Telegraph -> 115`。
- `查看资源 -> Telegraph -> 直写 ed2k`。
- `查看资源 -> Telegraph -> catbox 文本清单`。
- 动漫单集、剧集、电影、音乐/明星、合集卡片均可共用本规则。

## 处理流程（2026-04-05 优化）
1. **快速直判**：正文只要出现 `115cdn.com/s/`、`115.com/s/`、`magnet:?`、`ed2k://`，立即视为真实入口。
2. **立即执行官方命令**：
   - 115 分享链接 → `/p115_add_share`
   - 磁链 / `ed2k` → `/ol`
3. **只在必要时补查**：
   - 正文没有真实入口，才继续打开 `查看资源`
   - 命令执行失败，才回头补日志 / 页面 / 文档
4. **其他网盘链接**：默认不进入本地主链路，避免拖慢处理速度。

## 官方能力对齐
- 当前官方插件 `P115StrmHelper` 已确认支持：`/p115_add_share`、`/p115_share_strm`、`/ol`、`/sh`。
- TG卡片处理优先复用这些官方命令，不额外包一层自定义等待或验证步骤。
