# 115链接分流

此文档是 `P115StrmHelper` 链路的总入口，用来先区分“离线下载链接”和“115分享链接”，再决定该走哪条接口。

## 链接类型判定
- 离线下载链接：`ed2k://`、`magnet:?`、其他明确用于离线下载的直链。
- 115分享链接：`https://115.com/s/...`、`https://115cdn.com/s/...`、其他带提取码/密码的 115 分享页链接。

## 固定分流
1. 先判定链接类型，不跳过这一步。
2. 已判定为离线下载链接：直接走 `/ol <链接>`。
3. 已判定为 115 分享链接：默认走 `/p115_add_share <链接>`。
4. 少爷明确要求“分享链接直接生成 STRM”时：改走 `/p115_share_strm <链接>`。

## 快速判型
- 文本里出现 `115cdn.com/s/` 或 `115.com/s/`：直接判定为 115 分享链接。
- 文本以 `magnet:?` 或 `ed2k://` 开头：直接判定为离线下载链接。
- 文本里同时带 `访问码`、`提取码`、`password=`：若主链接是 115 域名，仍直接走 115 分享链路。
- 已满足以上任一条件时，不再额外读取说明文档、不再做页面验证，直接交插件命令。

## 执行规则
- 类型判定只做轻量分流，不做多余前置校验。
- 一旦分流完成，直接交给对应接口处理，不额外人工介入。
- `115` 分享链接不能误投 `/ol`。
- `/ol` 只处理离线下载类链接；`/p115_add_share` 只处理 115 分享链接；`/p115_share_strm` 仅在少爷明确要求分享直生 STRM 时使用。
- 离线下载与转存两条链路都按“直接交接口”执行，速度优先。
- 当少爷直接发送单个可判型真链时，默认本轮立即执行对应官方命令，不再先做站点检查、页面探测、文档补读、额外解释或确认。
- 只有命令执行失败、返回异常，或链接类型仍无法判断时，才进入补查和回退链路。
- 该文档是 115 相关下载/转存分流的唯一入口，其他骨架文件只做索引，不重复堆规则。
- 链接类型仍无法判断时，再向少爷确认，不自作主张。

## 官方能力对齐
- `P115StrmHelper` 官方命令能力：`/p115_add_share`、`/p115_share_strm`、`/ol`、`/sh`。
- 对已明确判型的 115 / magnet / ed2k 链接，直接使用对应官方命令，不增加自定义中间步骤。
- 只有命令执行失败、返回异常，或链接判型不明确时，才进入补查和回退链路。

## 例子
- `ed2k://...` -> `/ol`
- `magnet:?xt=...` -> `/ol`
- `https://115cdn.com/s/...?...` -> `/p115_add_share`
