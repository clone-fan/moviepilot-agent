# 本次整理归档

## 背景
本轮连续暴露出两类问题：
1. 搜索展示链路不完全符合用户固定要求（海报缺失、系列续作混入主条目优选）
2. 排查 `/sh` 与搜索日志时存在证据口径不严的问题，曾混入上一轮日志并导致错误结论

## 已落地修改

### 1. 搜索技能修复
文件：`/config/agent/skills/moviepilot-cli/SKILL.md`

已新增/重写规则：
- 搜索时若媒体有 `poster_path`，默认发送海报图，不再只发纯文本卡片
- 搜索结果先做“标题一致性与系列过滤”，再做优选资源筛选
- 明确过滤或降权：`WGP`、`MAX`、`Shippuuden`、`疾风传`、`Boruto`、`博人传`、剧场版、SP、OVA、合集等非当前主条目结果
- 图片发送失败时必须明确告知链路异常，不能假装已按图文卡片发出
- 废弃旧规则：`search_torrents` 后全量展示筛选项、`get_search_results` 后全量展示结果、允许续作混入当前主条目优选、图片链路可用却只发纯文本

### 2. 真实性约束修复
文件：
- `/config/agent/USER_PREFERENCES.md`
- `/config/agent/AGENT_PROFILE.md`
- `/config/agent/AGENT_HOOKS.md`
- `/config/agent/AGENT_WORKFLOW.md`

已新增/强化规则：
- 排查 `/sh`、插件命令、搜索日志、当前轮触发状态时，必须先核对本轮实际命令与对应日志
- 禁止拿上一轮搜索结果、模糊关键字日志或推测替代事实
- 涉及日志、命令、插件触发、异常归因、状态判断时，必须以本轮已核实证据为准
- 执行中必须先锁定“本轮时间窗 + 本轮关键词 + 本轮命令”再下结论
- 若证据不足，只能汇报“未确认/待确认”，不能脑补或硬下结论
