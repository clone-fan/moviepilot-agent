# 唤醒固定模板

## 1. 模板说明
此文件是手动唤醒与 heartbeat 唤醒共用的唯一模板与字段口径来源。其它文件只保留触发入口、流程位置与高层边界，不再重复保存同一套版式细节。

## 2. 固定顺序
1. **打招呼**：按当前时段实时生成问候语
2. **当前时间**：系统实时时间展示
3. **MoviePilot**：版本信息与更新状态
4. **站点增量**：当日站点数据相对昨日的增量
5. **今日入库**：当日成功转移的媒体记录
6. **今日追新**：当日订阅更新内容
7. **当前下载**：正在下载的任务状态

## 3. 固定格式
```text
少爷，早/午/晚上好～

### 当前时间
- YYYY-MM-DD HH:MM

### MoviePilot
- 当前版本：x.x.x
- 最新版本：x.x.x

### 站点增量
- 站点名：↑ x.xx GB | ↓ x.xx GB

### 今日入库
- N 个 / 或列 1-3 条代表项

### 今日追新
- N 部 / 或列 1-3 条代表项

### 当前下载
- N 个任务 / 或列 1-3 条活跃任务
```

## 4. 字段口径
### 4.1 打招呼
- **生成规则**：按当前时段实时生成，语气短、直接、自然，不使用旧人格口吻
- **时段对应**：凌晨(00-05)、早上(06-11)、下午(12-17)、晚上(18-23)

### 4.2 当前时间
- **数据源**：使用当前系统实时时间
- **展示格式**：`YYYY-MM-DD HH:MM`

### 4.3 MoviePilot
- **当前版本**：固定读取 `/api/v1/system/global?token=moviepilot` 的 `data.FRONTEND_VERSION` / `data.BACKEND_VERSION`
- **最新版本**：固定读取 GitHub Releases 接口并仅取 `tag_name` 为 `v2.*` 的最新版本
- **未取到处理**：若未取到则如实标记“暂无”

### 4.4 站点增量
- **数据源**：读取站点用户数据表，比较当日最新快照与前一日快照
- **展示格式**：`站点名：↑ 2.15 GB | ↓ 0.87 GB`
- **无昨日基线**：若缺少前一日数据，则不展示该站点
- **无数据处理**：没有记录统一写 `暂无`

### 4.5 今日入库
- **数据源**：以当日自然日（CST 00:00:00 至当前时刻）的 `query_transfer_history` 成功记录为准
- **展示格式**：`N 个` 或列 1-3 条代表项

### 4.6 今日追新
- **优先数据源**：优先读取 `/config/agent/docs/subscribereminder_last_push.json` 的当日结果
- **回退方案**：若文件缺失、过期或为空，再回退到其它 MoviePilot 接口方案，并明确标记回退

### 4.7 当前下载
- **数据源**：读取 `downloading` 任务
- **无任务处理**：没有任务统一写 `暂无`

## 5. 执行约束
1. **链路统一**：手动唤醒与 heartbeat 唤醒必须共用本模板与同一条根层实现链路
2. **执行顺序**：先接管，再取数，再按本模板渲染播报
3. **格式保持**：只修字段值，不改段落顺序与排版结构
4. **数据真实**：若字段未取到，必须如实写 `暂无`，不得复用旧结果
5. **TG发送约束**：若播报需要直达 Telegram，必须使用已绑定的 `telegram_userid` 作为明确目标，不能假设管理员默认可达
6. **规则同步**：规则修改后必须同步写回当前根层持久层相关文件
7. **主动汇报**：遇到问题、卡进度、完成关键项时，必须主动汇报
