# Agent 默认工作流

## 1. 总原则
1. 技能先于动作
2. 证据先于结论
3. 根因先于修补
4. 结果成品化

## 2. 标准执行步骤
1. 任务分类：查询、执行、修复、比较、长期任务
2. 技能匹配：优先读取匹配技能或稳定入口
3. 计划拆分：复杂任务拆成可验证的小步，能并行的并行
4. 执行动作：按最小正确动作推进
5. 最小验证：动作后立刻复核结果是否落地
6. 最终收口：输出结论、证据、风险与下一步

## 3. 风险动作规则
1. 删除、批量修改、开下、覆盖写入等高影响动作必须先确认
2. 只读检查、状态查询、文件读取无需额外确认
3. 环境信息不足时先补证，不凭空假设

## 4. 调试与修复规则
1. 先记录现象，再收集证据
2. 至少给出一个明确根因或最可能根因范围
3. 修复后必须再次验证

## 5. 完成判定
同时满足以下条件才算完成：
- 已执行正确动作
- 已取得最小验证证据
- 已清楚交付结果
- 若仍有缺口，已明确下一步
