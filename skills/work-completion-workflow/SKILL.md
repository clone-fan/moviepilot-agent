# 工作完成标准流程

## 工作流步骤

### 1. 完成工作
- 按照用户要求完成任务
- 记录修改的文件和改动内容

### 2. 申请检查bug
- 向用户申请检查bug
- 说明做了什么、改了哪些文件

### 3. 检查bug（工程师级别）
**我自己检查，不由用户检查**

检查标准：
- ✓ 通过多个实际案例验证 — 不只是口头验证
- ✓ 规则理解正确 — 确认掌握了核心规则
- ✓ 能正确识别和解析 — 实际操作验证
- ✓ 能正确调用对应技能 — 确认技能可用
- ✓ 技能已娴熟于心 — 确认可以立即执行

### 4. 申请执行整理
- 检查完毕，向用户申请执行整理流程

### 5. 执行整理流程
**用户同意后执行**

整理内容：
- **归纳实现** — 总结做了什么、改了哪些文件
- **归档资料** — 整理相关文档和日志，事件型内容优先归档到 `/config/agent/docs/archive/`
- **压缩日志** — 清理冗余和过期记录
- **整理记忆** — 只保留稳定偏好、行为边界与长期有效规则，不把单次事件过程写进长期记忆文件

### 6. 汇报操作
- 向用户汇报修改了什么文件
- 说明改动内容

### 7. 写工作日记
- 记录本次工作的完整过程
- 更新相关文档的执行日志

## 完成标志
汇报"已修正，流程完成"

## 适用范围
- 代码修改工作
- 文件整理工作
- 技能掌握工作
- 所有需要完整交付的工作
