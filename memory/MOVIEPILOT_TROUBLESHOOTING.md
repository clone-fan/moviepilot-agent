# MoviePilot 故障排查矩阵

## 原则
1. 先复现或观测，再修复。
2. 先定位链路断点，再修改配置或文件。
3. MoviePilot 问题优先按：站点 -> 识别 -> 资源 -> 下载 -> 转移 -> 媒体库 -> 通知 排查。

## 搜不到资源
- 首查：站点启用状态、站点连通性、搜索范围、媒体识别结果。
- 次查：标题、年份、类型、别名、自定义识别词、站点索引返回。
- 修复：换关键词、修识别词、扩大站点范围、检查 Cookie/UA、重新搜索。

## 下载失败或无速度
- 首查：下载器状态、下载任务状态、资源做种/促销/站点来源。
- 次查：保存路径、下载器连接、站点认证、任务标签。
- 修复：换资源、调整下载器或保存路径、检查站点认证、重新添加任务。

## 订阅不更新
- 首查：订阅状态、季集、开始集、总集数、订阅站点范围。
- 次查：缺集搜索、过滤规则、媒体库是否已存在、站点是否可用。
- 修复：`search_subscribe`、更新订阅过滤/季集、修正识别、调整站点范围。

## 转移失败
- 首查：`query_transfer_history` 的失败原因、源文件路径、目标目录。
- 次查：媒体识别、季集、权限、存储配置、历史记录是否阻塞重试。
- 修复：必要时 `delete_transfer_history` 后 `recognize_media` / `transfer_file`。

## 媒体库缺失
- 首查：下载是否完成、转移历史是否成功、媒体库是否已存在。
- 次查：刮削、命名、目录配置、媒体服务器同步状态。
- 修复：重新转移、刮削元数据、触发媒体库同步或检查服务器。

## 心跳或通知异常
- 首查：脚本编译、脚本退出码、通知发送日志。
- 次查：真实发送文本、用户绑定、通知渠道、数据源路径。
- 修复：修渲染、修路径、修通知绑定、手动触发后反查日志。
