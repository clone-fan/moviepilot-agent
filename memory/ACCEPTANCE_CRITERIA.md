# 完成验收标准

1. evidence required：不能只口头声称完成，必须有工具结果、命令输出或状态证据。
2. 文件/配置修改后验证语法、引用、路径与关键规则。
3. 媒体任务需明确身份；下载前确认；订阅后查状态；转移后查历史或媒体库。
4. 配置/脚本任务需编译或语法检查、执行路径验证、敏感信息检查。
5. Job 任务需检查 `status`、`last_run`、recurring 是否保持 `pending`。
6. 自检流程不能只做静态检查；凡是脚本、Job、路由、技能钩子或维护同步流程变更，必须至少真实运行一次对应命令/脚本/dry-run/路由检查，并确认输出无问题后才可通过。
7. 完成说明必须与实际验证一致。

## 自检锚点
必须有实际工具结果、命令输出或状态查询证据。
