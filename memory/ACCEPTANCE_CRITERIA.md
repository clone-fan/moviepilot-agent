# 完成验收标准

1. evidence required：不能只口头声称完成，必须有工具结果、命令输出或状态证据。
2. 文件/配置修改后验证语法、引用、路径与关键规则。
3. 媒体任务需明确身份；下载前确认；订阅后查状态；转移后查历史或媒体库。
4. 配置/脚本任务需编译或语法检查、执行路径验证、敏感信息检查。
5. Job 任务需检查 `status`、`last_run`、recurring 是否保持 `pending`。
6. 自检流程不能只做静态检查；凡变更可执行流程，必须真实运行对应最小验证，不只做静态检查。
7. Agent 自身治理任务需验证减负结果：常驻内容是否减少、重复规则是否合并、低频流程是否迁移、关键边界是否保留。
8. 完成说明必须与实际验证一致。
9. Agent 能力、插件、工作流或治理类变更的完成证据应能压缩成 `Command/Tool -> Output/State -> Claim`，且声明范围不得超过证据覆盖范围。
10. 运行时影响类变更不能只靠静态 grep；至少需要一次代表性 dry-run、路由检查、命令发现、安全实跑或权威状态查询。

## 自检锚点
必须有实际工具结果、命令输出或状态查询证据。
