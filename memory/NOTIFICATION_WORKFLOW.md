# 通知与心跳工作流

## 主动通知场景
1. 用户明确要求通知。
2. 心跳播报。
3. 高风险操作完成或失败。
4. 下载、订阅、转移、站点异常需要提醒时。

## 心跳规则
1. 心跳模板固定，数据只允许真实状态变化。
2. 心跳业务数据来自确定性脚本或 MoviePilot 内部路径。
3. AI 不生成、猜测或补全心跳业务数据。
4. 手动触发心跳后必须从发送日志反查实际通知内容。

## 通知失败排查
- 用户绑定是否存在。
- 通知渠道是否启用。
- `post_message` 是否执行。
- 日志是否有发送记录。
- 文本是否存在乱码、旧字段或占位数据。
