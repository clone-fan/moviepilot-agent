# Agent 目录治理规则

## 职责矩阵
- `docs/`：历史资料、归档、旧版记忆、回溯查询；不作为现行系统数据源。
- `jobs/`：Job 定义、状态、`last_run`、执行日志；每个 Job 目录只放 `JOB.md`。
- `scripts/`：可执行维护脚本、确定性工具脚本。
- `runtime/`：当前运行态数据、缓存、当前生效配置与 persona。
- `memory/`：长期规则、偏好、稳定边界；只放直接可加载的 `.md` 规则文件。
- `skills/`：Agent 技能定义及技能私有辅助文件。
- `activity/`：系统自动活动日志，只读，不手动维护。

## 放置原则
1. 文件放在哪里，由它“是什么”决定，不由“谁使用它”决定。
2. 缓存是运行态，放 `runtime/`；脚本是执行体，放 `scripts/`；Job 是调度定义，放 `jobs/`；历史资料才放 `docs`；用户长期表达偏好才放 `USER_PREFERENCES.md`。
3. 新增或迁移文件前，先判断职责归属，再做最小正确改动。

## 禁止事项
1. 禁止 `docs/` 作为现行系统数据源。
2. 禁止 `jobs/` 放脚本、缓存、临时文件或非 `JOB.md` 文件。
3. 禁止把运行态缓存放入 `docs/` 或 `memory/`。
4. 禁止把单次执行流水写入长期记忆；执行历史应进入 Job 日志或 activity。
5. 禁止把临时审计文件沉淀到 `docs/`。
6. 禁止修改官方/内置 skills 来承接本地偏好；本地增强应独立成 skill 或脚本。
7. 禁止把技能流程、交互模板、业务规则或一次性修正塞进 `USER_PREFERENCES.md`。

## 变更流程
1. 判断文件职责。
2. 放入对应目录。
3. 更新所有现行引用。
4. 扫描旧路径，区分现行引用与 `docs/` 历史引用。
5. 编译、执行或渲染验证。
6. 清理 `__pycache__`、临时文件、备份文件。
7. 只把稳定规则写入 `memory/`，不要把单次过程写入长期记忆。

## 自检基线
- `docs/` 根目录仅保留入口说明和归档目录。
- `jobs/*/` 只保留 `JOB.md`。
- `scripts/` 可放确定性维护脚本，但应避免残留 `__pycache__`。
- `runtime/cache/` 承载当前运行缓存。
- 现行系统引用不得指向 `docs/` 中的运行态旧路径。
